The best proviers for Big Data analysis are Google Cloud and Amazon Web Services

Amazon Web Services - RedShift

  • If your company uses Amazon Simple Storage, RedShift Spectrum enables you to run RedShift queries directly against Amazon S3 storage, which is excellent for accessing your data lakes. If so, RedShift Spectrum is almost certainly a superior choice for your company.
  • You have the option of using RedShift's huge Dense Storage or its dense compute. You can spin up a 160GB dc2.large node for $0.25 per hour, which is the least expensive node you can buy. Dense Storage costs $4.25 per TB per hour to operate
  • This makes using RedShift interesting. You can significantly reduce costs if you can predict your run-times and how frequently you'll need to spin up each node, especially if you pay in advance.

Google Cloud - BigQuery

  • Google will offer more services than AWS does, this gives Google the advantage in the Big Data segment, but we are just focusing the analysis on Data Warehouse
  • BigQuery is suitable for heavy queries, those that operate using a big set of data. The bigger the dataset, the more it is likely to gain in performance. This is when compared to the traditional relational databases BigQuery implements different parallel schemas to speed up the execution time.
  • Considering the pricing for the service if you run rapid queries a few times a day, you do data mining operations or you are a data scientist running ML BigQuery would be a far better option

Conclusions

There isn't a winner here. RedShift is more economical for everyday data warehouse operations for most businesses. But, BigQuery is better for businesses looking to do data mining or those who deal with extremely variant workloads.
